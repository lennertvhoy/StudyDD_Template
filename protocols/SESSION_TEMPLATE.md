# SESSION_TEMPLATE — Standard Study Session Structure

> **Agent-maintained.** Use this template to keep sessions consistent and auditable.

## Before the session

1. Read `AGENTS.md`.
2. Read `state/STUDY_STATUS.md`, `state/STUDY_STATE.yaml`, `state/NEXT_STUDY_ACTIONS.md`, `state/STUDY_BACKLOG.md`, `state/SKILL_MAP.yaml`.
3. Confirm the active focus with the learner.
4. Select the next best question based on weak areas and pending validation.

## During the session

For each question:

1. State the active question ID.
2. State the expected answer format.
3. Ask exactly one question.
4. Wait for the learner's answer.
5. Grade against the answer key.
6. Explain the verdict.
7. If needed, ask one repair question.
8. Close the question before asking the next.

End the session after a planned number of questions or when the learner asks to stop.

## After the session

1. Summarize what was covered.
2. List new evidence items.
3. Identify weak areas.
4. Propose updates to `state/SKILL_MAP.yaml`.
5. Propose updates to `state/STUDY_STATE.yaml`.
6. Propose updates to `state/NEXT_STUDY_ACTIONS.md`.
7. Propose updates to `state/STUDY_STATUS.md`.
8. Append entries to `state/SESSION_LOG.md` and `state/EVIDENCE_LOG.md`.
9. Wait for learner confirmation or apply authorized updates.
10. End with a clear next action.

## Session length

Default: 3–5 exam-style questions per session.

Adjust based on learner preference, available time, and topic difficulty.
